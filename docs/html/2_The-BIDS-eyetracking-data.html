The BIDS eyetracking data¶

The tree structure of this data is as follows

dataset /

dataset_description.json
participants.tsv
participants.json

sub-<label> /
[ses-<label> /]
<datatype> /
<filename> _eyetrack.tsv.gz
<filename> _eyetrack.json
<filename> _events.tsv
<filename> _events.json

For more details on the tree structure of this data, see the BIDS documentation.

Exemple of the BIDS eyetracking data¶

Here we present an example of a data tree

In this example we have eye movement data from two subjects (sub-001 and sub-002).
This data comes from the Eyelink tracking system.
The first subject had one session and the second had two.

Description of the files¶



dataset_description.json¶

The file dataset_description.json is a JSON file describing the dataset.
See the documentation for more details.

For exemple :¶
{
    "Name": "SuperTache",
    "BIDSVersion": "1.8.1"
}


participants.tsv¶

The file participants.tsv describe the properties of the participants such as GUID, age, sex, and handedness.
See the documentation for more details.

For exemple :¶
participant_id GUID
001 3aabe8bb-b5dd-4934-aeaa-cda1e0026609
002 5b2ee5b9-549b-4624-88b9-897295f3e5a0


participants.json¶

The participants.json file documents the columns present in the participants.tsv file.
See the documentation for more details.

For exemple :¶
{
    "GUID": {
        "Description": "globally unique identifier"
    }
}


filename_eyetrack.asc¶

The file <filename> _eyetrack.asc is the raw data file from the Eyelink tracking system.
